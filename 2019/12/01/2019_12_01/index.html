「服务设计方法和工具」XXY创新工作坊回顾

2019-12-01 PV:

前言

本篇是回顾年中时参加的由普禾资本旗下研究院与XXY Innovation组织的「智能语音语义与图像技术应用创新工作坊」，工作坊围绕 声希科技的语音、语义及图像等核心技术，利用设计思维等相关工具和方法论，开展技术创新应用及商业模式的研讨。本次研讨以PM身份作为被邀对象，与业界技术专家、投资人、设计总监和教授等，在XXY Innovation创始人、著名设计学者辛向阳教授的指导下，参与技术产品化的讨论。本文将主要聚焦在服务设计方法和工具的介绍与应用上：

  • INPD整合新产品开发流程
  • VOA价值机会分析
  • 产品属性分析
  • 利益相关者分析
  • 路径九宫格

声希科技，来自香港中文大学人机交互实验室师生团队，目前业务方向主要聚焦在2B教育以及娱乐领域。

iNPD整合新产品开发流程

1）简介与阶段
iNPD是Jonathan Cagan与Craig Vogel教授建立的一种以用户为中心，充分考虑用户需求、利益相关者诉求的产品开发流程，拥有稳定的理论基础与系统的方法体系，适用于跨职能、多背景的产品团队写作。在从产品策划到项目批准的iNPD程序中主要包括四个阶段：

  • 阶段一：确定产品方向
  • 阶段二：用户研究分析
  • 阶段三：产品概念定义
  • 阶段四：产品实施落地

2）确定产品方向
在阶段一中，通过分析企业的外部环境，会用到[^SET分析法]，SET分析法早期是由PEST分析方法转变而来，在实战应用中，工作人员发现针对人的研究应更深入，因而转变为SET。SET分析法就是使用系统工具进行「社会、经济、技术」三个环境分析之后，通过宏观、中观、微观三个层面进行精准地定义市场潜在需求和企业机会的方法论。

社会趋势 经济动力 技术趋势
宏观 国家政策 世界经济 全球技术
中观 居民生活 产业经济 技术应用
微观 市场需求 行业经济 生产使用

投影查询：select <字段1>，<字段2>，<字段3> from <表名>

排序查询：select * from <表名> ORDER BY <字段1>，<字段2>

分页查询：LIMIT OFFSET 截取 第m条到n条的数据

聚合查询：

  • COUNT() 某列个数
  • SUM () 某列合值
  • AVG () 某列均值
  • MAX () 某列最大
  • MIN () 某列最小

多表查询：

  • SELECT * FROM <表1>, <表2> 返回的为两表乘积
  • SELECT <表1.旧字段 新字段>,<表2.旧字段 新字段> FROM <表1>, <表2> ; 返回多个某表某字段段数据，替换字段名
  • <表名/列名> +<空格>+ <替换名> 返回的数据以替换名呈现

连接查询：

  • FROM <表1> INNER JOIN <表2> 两张表都存在的记录
  • FROM <表1> LEFT OUTER JOIN <表2> 将表2的数据加到表1上
  • FROM <表1> RIGHT OUTER JOIN <表2> 将表1的数据加到表2上
  • FROM <表1> FULL OUTER JOIN <表2> 选出两表都存在的记录

增、删、改

增：INSERT INTO <表名> (字段1, 字段2, …) VALUES (值1, 值2, …)

删：DELETE FROM <表名> WHERE …UPDATE：更新已有记录

改：<表名> SET 字段1=值1, 字段2=值2, … WHERE …
