使用Hexo搭建博客

2019-11-29 日常分享 PV:

前言

第一次做技术类分享，本次博客搭建前断断续续花了3天时间。最早在整理文档和笔记时陆续使用过有道云笔记、印象笔记、备忘录等，但是主要应用场景还是在内容的摘录、处理事项和文档保存方便查阅的场景上，对于存粹自己写一些东西分享出来就比较少。

前几天无意中在逛Youtube的时候了解到利用Github可以直接搭建个人博客站点，而且供大家使用的博客框架有很多，比如我用到的Hexo，其他知名还有Jekyll、Wordpress等等。

为什么选择Hexo

1）主流博客搭建方案

  1. Git+Github+Markdown+jekyll （免费）
  2. Git+Github+Markdown+hexo （免费）
  3. 虚拟主机＋插件＋Wordpress （付费）

2）难度系数

hexo<jekyll<Wordpress。

优点：

  • 静态加载速度快；
  • 支持Markdown编辑；
  • 免费部署在Github上；
  • 命令比Jekyll简单。

缺点：

  • 需要一定网页基础；
  • 依赖本地环境，更换电脑需要重新配置。

3）小结
对于大部分像我这样的小白来说，刚开始会选择用前2种方案，首先是免费，其次付费还需要备案，新手门槛较高，当然对于后期进阶我也会尝试第3种方案。

常见问题

1）终端返回4000端口被使用，这个时候可以自定义换一个端口，运行：
、、、bash
sudo hexo server -p 5000
、、、
另外，每次使用新端口，避免再出现端口被使用，结束时执行 Citrl+C。

2）Hexo主题中的图片替换，一般在themes/source/css/images路径下，支持的文件格式有jpg/png/svg等。

3）在用atom编辑器时，需要系统分配文件修改权限，打开Mac下终端Terminal，执行c d打开需要修改文件的上一级目录，再执行：
、、、bash
sudo hexo server -p 5000
、、、sudo chmod -R 777 <文件名>
